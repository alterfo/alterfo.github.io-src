# KODA.md — Контекст проекта

## Обзор проекта

Это персональный блог Олега Сидоркина, построенный на **VuePress 1.x**. Проект включает в себя:

1. **Блог** с постами на различные темы (разработка, управление, личностный рост)
2. **IDEF0 Editor** — интерактивный визуальный редактор для создания IDEF0 диаграмм

### Основные технологии

| Технология | Версия | Назначение |
|------------|--------|------------|
| VuePress | 1.9.10 | Статический генератор сайтов |
| Vue.js | 2.x | Фреймворк для компонент |
| Yarn | 4.7.0 | Менеджер пакетов |
| IndexedDB | - | Клиентское хранилище для IDEF0 Editor |

### Структура проекта

```
.
├── blog/                      # Исходный контент блога
│   ├── _posts/               # Посты блога (Markdown)
│   ├── .vuepress/            # Конфигурация VuePress
│   │   ├── config.js         # Основной конфиг
│   │   ├── components/       # Vue-компоненты
│   │   │   └── IDEF0Editor/  # Модули редактора IDEF0
│   │   ├── theme/            # Кастомная тема
│   │   └── public/           # Статические ресурсы (WebGPU шейдеры)
│   ├── about.md              # Страница "О проекте"
│   └── idef0.md              # Демо-страница редактора
├── package.json              # Зависимости и скрипты
├── deploy.sh                 # Скрипт деплоя
└── README.md                 # Документация проекта
```

## IDEF0 Editor — Детали реализации

### Назначение

Визуальный редактор для создания IDEF0 (Integration Definition for Function Modeling) диаграмм — стандарта функционального моделирования.

### Ключевые функции

| Функция | Описание |
|---------|----------|
| Создание блоков | Добавление прямоугольных блоков с текстом |
| Стрелки ICOM | Input, Control, Output, Mechanism — 4 типа связей |
| Декомпозиция | Вложенные диаграммы при клике на блок |
| Панорамирование | Колесо мыши или Alt+ЛКМ |
| Масштабирование | Колесо мыши (зум к курсору) |
| Редактирование текста | Двойной клик по блоку/стрелке |
| Экспорт | PNG, SVG, JSON |
| Автосохранение | IndexedDB с синхронизацией между вкладками |

### Архитектура редактора

```
IDEF0Editor.vue (основной компонент)
├── constants.js   # Константы (цвета, размеры)
├── db.js          # IndexedDB абстракция
├── validation.js  # Валидация диаграмм
├── exporter.js    # Экспорт в форматы
├── router.js      # Управление URL
├── hierarchy.js   # Вложенность диаграмм
├── renderer.js    # Canvas отрисовка
├── manhattan.js   # Manhattan routing для стрелок
├── minimap.js     # Миникарта
└── ...
```

### Структура данных диаграммы

```javascript
{
  id: 'A0',
  name: 'Контекстная диаграмма',
  blocks: [
    {
      id: 'block-1',
      name: 'Функция',
      x, y, w, h,
      diagramId: null // ID вложенной диаграммы
    }
  ],
  arrows: [
    {
      id: 'arrow-1',
      name: 'Поток',
      type: 'input|output|control|mechanism',
      from: { blockId, edge, offset },
      to: { blockId, edge, offset },
      segments: [] // Для изогнутых стрелок
    }
  ],
  view: { x, y, scale }
}
```

### Цветовая схема стрелок

| Тип | Цвет |
|-----|------|
| Input | #2980b9 (синий) |
| Output | #27ae60 (зелёный) |
| Control | #e74c3c (красный) |
| Mechanism | #8e44ad (фиолетовый) |

## Правила разработки

### Стиль кода

- **Vue components**: Single File Components (`.vue`)
- **Naming**: PascalCase для компонентов, camelCase для методов
- **Комментарии**: Русский язык в комментариях
- **Modularity**: Разделение по функциональности в отдельных файлах

### Тестирование

- Файлы тестов имеют суффикс `.spec.md` (например, `IDEF0Editor.spec.md`)
- Используется Vue Test Utils

### Контрибуция

1. Создавайте функциональные ветки
2. Пишите комментарии к сложным участкам кода
3. Тестируйте изменения в режиме разработки (`yarn dev`)
