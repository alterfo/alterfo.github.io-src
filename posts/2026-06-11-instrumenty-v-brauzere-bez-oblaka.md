Запись в Notion — это текст на чужом сервере. Задачи в облачном планировщике — данные в чужой аналитике. "Бесплатный" инструмент в 2026-м — это вопрос к бизнес-модели, а не к доброй воле создателей.

Волна локальных LLM показала широкой аудитории, что on-device — это реально: модель работает, данные не покидают машину. Тот же сдвиг происходит с обычными инструментами: всё заметнее, что ежемесячная подписка — это аренда софта, которым не владеешь и который нельзя отключить от сети.

Семь приложений на этом сайте — попытка пройти этот путь до конца. Все работают целиком в браузере: без сервера, без регистрации, без отправки данных куда-либо. Только локальное хранилище, шифрование на стороне клиента и нативные браузерные API. Идея "колеса жизни", вокруг которого они собраны, описана ещё в 2020-м — [Круг жизни](/posts/2020-09-04-canvas-circle-of-life); тогда это была визуализация, теперь за каждым сегментом стоит работающий инструмент.

## IDEF0-редактор — функциональные диаграммы, которые никуда не отправляются

Диаграммы IDEF0 описывают бизнес-процессы изнутри. Когда компания рисует схему "как мы обрабатываем заявки" или "как устроена цепочка согласований" — это внутренняя кухня, которую не хочется отправлять в чужое облако. При этом специализированного браузерного редактора IDEF0 до сих пор не было вообще: стандарт нишевой, и в универсальных рисовалках он превращается в набор прямоугольников без семантики.

Редактор написан на SVG + Vue 3, строго по американскому федеральному стандарту FIPS 183. Есть валидация ICOM-правил: у каждой стрелки строгая семантика — вход (слева), управление (сверху), выход (справа), механизм (снизу). Нарушение правил подсвечивается прямо на диаграмме, как ошибка компилятора. Диаграммы складываются в иерархию: крупный блок можно "провалить" вглубь и расписать подробнее — стандартный паттерн системного анализа.

Всё хранится в IndexedDB браузера, синхронизируется между вкладками через localStorage. Экспорт: SVG, PNG (2×), JSON с полным round-trip. Предыдущая версия была на canvas, где нельзя выделить текст, — при полном переписывании от canvas отказались.

## Зашифрованный дневник в браузере

Большинство дневниковых приложений хранят записи в облаке разработчика и берут за это подписку: Day One синхронизирует через свои серверы, Notion индексирует всё, что в него попадает. Для текста, который пишется только для себя, это странная архитектура.

Здесь дневник устроен иначе: шифрование AES-GCM, ключ выводится из пароля через PBKDF2 в 600 тысяч итераций и существует только в памяти открытой вкладки. Вкладка закрыта — с устройства взять нечего. В IndexedDB лежит только зашифрованный конверт: `{ salt, iterations, iv, ciphertext }`, всё в base64. Открытого текста на диске нет, никогда.

Встроена механика 500 слов в день: счётчик прогресса и стрик. Просмотр прошлых записей — в том же интерфейсе, расшифровка происходит только после ввода пароля.

Синхронизация между своими устройствами — через WebRTC DataChannel, без сервера, без STUN/TURN, в локальной сети. Обмен offer/answer кодами идёт через любой удобный канал, после чего зашифрованные конверты передаются напрямую и расшифровываются на каждой стороне отдельно. У каждого устройства свой salt — дополнительный барьер поверх DTLS-шифрования самого канала. Вне локальной сети — перенос зашифрованным файлом.

## Тренажёр фортепиано с MIDI-клавиатурой

Для разучивания пьес нужна мгновенная обратная связь: та нота сыграна или не та. Synthesia делает это давно, но это нативное приложение, которое надо ставить и покупать.

Web MIDI API нативно поддерживается в Chrome и Edge: браузер подключается к MIDI-клавиатуре без библиотек и плагинов. На этом построен тренажёр: OpenSheetMusicDisplay рендерит фразу через MusicXML, сыгранное сверяется с партитурой по MIDI-событиям в реальном времени. Два режима: повтор такта (для разучивания) и проигрыш фразы целиком (для шлифовки). Синтез — Tone.js, для тех, у кого нет физической клавиатуры; опционально подключаются семплы Salamander Grand Piano (отдельные статические файлы, не в JS-бандле).

Импорт своих пьес: MusicXML, ABC-нотация, MIDI-файлы. MIDI-импорт читает тактовый размер из файла; если размера там нет — приложение спрашивает его перед разбором.

Safari не поддерживает Web MIDI API, Firefox — только под флагом в настройках. Это нативный API: полифилом без установки его не заменить.

## Редактор поз OpenPose для ControlNet

В работе со Stable Diffusion ControlNet принимает скелет OpenPose как условие для управления позой персонажа. Обычно скелет готовят в AUTOMATIC1111 или ComfyUI — это локальные Python-инструменты с отдельной установкой. Онлайн-редакторы на Hugging Face работают через их серверы; изображения туда уходят.

MediaPipe BlazePose запускается в браузере как WASM — полностью локально, без серверных вызовов. Редактор обрабатывает очередь загруженных изображений, определяет до двух человек на кадре, конвертирует 33 ключевые точки BlazePose в 18 точек COCO OpenPose (шея вычисляется как середина плеч — в BlazePose такой точки нет, там плечи даются отдельно). Дальше точки перетаскиваются мышью, людей в кадре можно добавлять и убирать.

Экспорт: PNG с чёрным фоном для ControlNet + JSON OpenPose v1.3 с нормализованными координатами 0–1. Модель весит ~10,8 МБ и загружается при старте с этого же сайта — не с чужого сервера.

→ [OpenPose Editor](/openpose)

## Планировщик задач с шифрованием

База обычная: проекты, задачи, приоритеты, сроки, канбан-доска и список. Шифрование то же, что в дневнике, ключ живёт только в памяти сессии. Приватные заметки к задачам не покидают зашифрованное хранилище никогда.

Необычная часть — мост File System Access. Планировщик записывает в выбранную локальную папку plaintext `tasks.json`: структура данных без заметок. Агент вроде Claude Code может читать и редактировать этот файл прямо на диске — не зная пароля и не видя заметок. При следующем фокусе окна приложение перечитывает файл и сливает изменения по принципу last-write-wins: побеждает задача с большим `updatedAt`. Отсутствие задачи в файле не считается удалением — устаревший файл не сотрёт данные.

File System Access работает только в Chrome и Edge. В Firefox и Safari мост просто недоступен, без падений.

→ [Планировщик](/planner)

## Журнал решений

Самый нишевый из семи. Решение фиксируется в момент принятия: контекст, варианты, выбор, ожидаемый исход, уверенность в процентах, дата пересмотра. Когда дата наступает, приложение показывает карточку: сбылось или нет. Из накопленных ответов считается оценка Брайера (0 — идеальная калибровка, 0.25 — монетка) и таблица по корзинам уверенности: "заявлено 70% — сбылось X%".

Смысл не в угадывании, а в обратной связи: насколько субъективные оценки соответствуют реальности. Уверенность "на 90%" в событиях, которые сбываются в 60% случаев, — полезный факт о собственном мышлении.

Локальных аналогов практически нет: существующие инструменты для отслеживания решений — либо тетрадь, либо корпоративные платформы для командных ретроспектив. Приватный зашифрованный журнал с калибровочной статистикой — пустующая ниша.

## AR Engine — аудиовизуализатор на WebGPU

Единственный из семи, где код выходит за пределы JavaScript. Задача: не просто слышать звук, а видеть его структуру — где бас, где атака, насколько "яркий" тембр в данный момент.

Аудиоанализ написан на C и собран в WASM через Emscripten: FFT, четыре полосы RMS-огибающих (суб, бас, мид, высокие), детектор битов по спектральному потоку, спектральный центроид. JS читает структуру `AudioFrame` из 13 чисел прямо из WASM-кучи каждый кадр — без сериализации между слоями.

Рендеринг — compute shaders на WebGPU, до 500 тысяч частиц. На слабом железе число частиц автоматически снижается до 300k или 150k. Четыре режима: Fluid (плазма), Cymatics (узоры Хладни), Nova (зрачок, расширяющийся на каждый удар), Auto (переключает режимы по музыкальной сетке). Частотные полосы двигают разные вещи независимо — бас отдельно, высокие отдельно, атаки отдельно.

WebGPU начинался как "только Chrome 113+", за последний год подтянулись Safari 26 и Firefox на Windows. Самый предсказуемый вариант всё ещё Chrome. Там, где работает, это другой класс по сравнению с Canvas2D при любом количестве частиц.

---

Все семь инструментов сделаны для личного использования — отсюда их форма: без регистрации, без мультитенантности, без монетизации. Только браузерные API, шифрование на клиенте и данные, которые не уходят с устройства.

Честности ради: у local-first есть цена. IndexedDB — хранилище браузера, и браузер вправе его почистить. Поэтому в каждом инструменте есть экспорт в файл: бэкапы — ответственность пользователя, а не чьего-то облака.

Это не идеология, а инженерный размен: меньше чужих серверов — больше собственной ответственности.
